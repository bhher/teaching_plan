3. grid-area (영역 이름으로 배치)

grid-template-areas로 정의한 영역 이름으로 아이템을 배치합니다.

Header
Sidebar
Main Content
Aside
Footer
/* 컨테이너 */ .container { display: grid; grid-template-columns: 200px 1fr 200px; grid-template-rows: 100px 1fr 100px; grid-template-areas: "header header header" "sidebar main aside" "footer footer footer"; gap: 15px; } /* 아이템 */ header { grid-area: header; } .sidebar { grid-area: sidebar; } main { grid-area: main; } .aside { grid-area: aside; } footer { grid-area: footer; }
장점:
  • 시각적으로 레이아웃 구조를 파악하기 쉬움
  • 반응형 디자인에서 영역 순서 변경이 쉬움
  • 코드 가독성이 향상됨

4. grid-area 단축 사용법

grid-area는 grid-column/grid-row의 단축 속성으로도 사용할 수 있습니다.

/* grid-area 단축 */ .item { grid-area: 1 / 1 / 3 / 3; /* row-start / col-start / row-end / col-end */ } /* 위와 동일 */ .item { grid-row-start: 1; grid-column-start: 1; grid-row-end: 3; grid-column-end: 3; }
순서: row-start / col-start / row-end / col-end

5. 복잡한 영역 배치 예제

여러 기법을 조합한 복잡한 레이아웃입니다.

Header (2x1)
Sidebar (2x2)
Nav (1x2)
Main (2x2)
Aside (1x2)
Footer (3x1)
/* 복잡한 레이아웃 */ .container { display: grid; grid-template-columns: repeat(4, 1fr); grid-template-rows: repeat(4, 80px); gap: 10px; } .header { grid-column: 1 / 3; grid-row: 1 / 2; } .sidebar { grid-column: 3 / 5; grid-row: 1 / 3; } .nav { grid-column: 1 / 2; grid-row: 2 / 4; } .main { grid-column: 2 / 4; grid-row: 2 / 4; } .aside { grid-column: 4 / 5; grid-row: 3 / 5; } .footer { grid-column: 1 / 4; grid-row: 4 / 5; }
