목록 태그 예제

1. 순서 없는 목록 (ul, li)

항목의 순서가 중요하지 않은 목록에 사용합니다.

네비게이션 메뉴

  • 홈
  • 소개
  • 서비스
  • 연락처

기능 목록

  • 사용자 인증
  • 데이터 저장
  • 보고서 생성

코드 예시

<ul>
  <li>항목 1</li>
  <li>항목 2</li>
  <li>항목 3</li>
</ul>

2. 순서 있는 목록 (ol, li)

항목의 순서가 중요한 목록에 사용합니다.

레시피 단계

  1. 재료를 준비합니다
  2. 물을 끓입니다
  3. 면을 넣고 3분간 끓입니다
  4. 양념을 넣고 섞습니다
  5. 완성!

영문자 순서 (type="a")

  1. 첫 번째 항목
  2. 두 번째 항목
  3. 세 번째 항목

로마 숫자 순서 (type="I")

  1. 첫 번째 항목
  2. 두 번째 항목
  3. 세 번째 항목

시작 번호 지정 (start="5")

  1. 다섯 번째 항목
  2. 여섯 번째 항목
  3. 일곱 번째 항목

코드 예시

<ol type="a" start="3">
  <li>c 항목</li>
  <li>d 항목</li>
</ol>

3. 중첩 목록

목록 안에 목록을 넣을 수 있습니다.

웹 개발 학습 순서

  1. HTML 기초
    • 문서 구조
    • 시맨틱 태그
    • 폼 요소
  2. CSS 기초
    • 선택자
    • 박스 모델
    • 레이아웃
  3. JavaScript 기초
    • 변수와 함수
    • DOM 조작
    • 이벤트 처리

코드 예시

<ul>
  <li>메뉴 1
    <ul>
      <li>하위 메뉴 1-1</li>
      <li>하위 메뉴 1-2</li>
    </ul>
  </li>
  <li>메뉴 2</li>
</ul>

4. 정의 목록 (dl, dt, dd)

용어와 정의를 표현하는 목록입니다.

웹 기술 용어 사전

HTML
HyperText Markup Language의 약자로, 웹 페이지의 구조를 만드는 마크업 언어입니다.
CSS
Cascading Style Sheets의 약자로, 웹 페이지의 스타일을 정의하는 언어입니다.
JavaScript
웹 페이지에 동적인 기능을 추가하는 프로그래밍 언어입니다.
API
Application Programming Interface의 약자로, 소프트웨어 간 통신을 위한 인터페이스입니다.

FAQ 형식

Q: HTML과 CSS의 차이는 무엇인가요?
A: HTML은 웹 페이지의 구조를 만들고, CSS는 그 구조의 스타일을 꾸밉니다.
Q: JavaScript는 필수인가요?
A: 기본적인 웹사이트는 HTML과 CSS만으로도 만들 수 있지만, 동적인 기능을 위해서는 JavaScript가 필요합니다.

코드 예시

<dl>
  <dt>용어</dt>
  <dd>용어에 대한 설명입니다.</dd>
  
  <dt>또 다른 용어</dt>
  <dd>또 다른 용어에 대한 설명입니다.</dd>
</dl>

5. 목록 사용 가이드

  • ul (순서 없는 목록): 메뉴, 기능 목록, 체크리스트
  • ol (순서 있는 목록): 단계별 가이드, 순위, 레시피
  • dl (정의 목록): 용어 사전, FAQ, 메타데이터
