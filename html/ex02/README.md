# 문서 구성 요소와 콘텐츠 마크업 심화 예제

이 폴더는 HTML의 문서 구성 요소와 콘텐츠 마크업을 심화 학습하기 위한 예제 파일들을 포함합니다.

## 예제 파일 목록

### 1. `01-lists.html`
- 목록 태그 사용법
- 순서 없는 목록 (ul, li)
- 순서 있는 목록 (ol, li) - type, start 속성
- 중첩 목록
- 정의 목록 (dl, dt, dd)

### 2. `02-tables.html`
- 테이블 기본 구조 (table, tr, th, td)
- 테이블 섹션 (thead, tbody, tfoot)
- 셀 병합 (colspan, rowspan)
- 테이블 캡션 (caption)
- 테이블 접근성 (scope, headers 속성)

### 3. `03-forms.html`
- 다양한 input 타입 (text, email, password, number, date, tel, url, search)
- 체크박스와 라디오 버튼
- 선택 박스 (select, option, optgroup)
- 텍스트 영역 (textarea)
- 레이블 (label)과 연결
- 버튼 (button)
- 폼 유효성 검사 (required, pattern, minlength, maxlength 등)
- 폼 접근성 가이드

### 4. `04-media-embed.html`
- 오디오 태그 (audio)
- 비디오 태그 (video)
- 비디오 자막 (track)
- iframe 기본 사용법
- YouTube 비디오 임베드
- Google Maps 임베드
- iframe 보안 (sandbox 속성)
- 미디어 접근성

### 5. `05-microdata-aria.html`
- 마이크로데이터 기본 개념
- Schema.org 타입 (Person, Organization, Article)
- ARIA 기본 개념
- ARIA 속성 (role, aria-label, aria-labelledby, aria-describedby, aria-required, aria-invalid, aria-hidden, aria-live)
- ARIA 사용 가이드라인
- 마이크로데이터와 ARIA 비교

### 6. `06-complete-example.html`
- 모든 내용을 종합한 완성 예제
- 회원가입 폼 형태의 웹페이지
- 목록, 테이블, 폼, 마이크로데이터, ARIA를 모두 활용한 실제 예제

## 사용 방법

1. 각 HTML 파일을 웹 브라우저에서 열어보세요
2. 브라우저의 개발자 도구(F12)를 사용하여 소스 코드를 확인하세요
3. 각 예제의 코드를 수정해보며 학습하세요
4. 폼 예제는 실제로 입력해보며 유효성 검사를 테스트하세요

## 학습 체크리스트

각 예제를 학습한 후 다음 사항을 확인하세요:

### 목록
- [ ] ul, ol, dl을 적절히 사용할 수 있는가?
- [ ] 중첩 목록 구조를 이해했는가?
- [ ] 정의 목록의 용도를 이해했는가?

### 테이블
- [ ] thead, tbody, tfoot을 사용할 수 있는가?
- [ ] th 태그와 scope 속성을 올바르게 사용할 수 있는가?
- [ ] caption을 사용할 수 있는가?
- [ ] 셀 병합(colspan, rowspan)을 이해했는가?
- [ ] 테이블 접근성을 고려할 수 있는가?

### 폼
- [ ] 다양한 input 타입을 사용할 수 있는가?
- [ ] label과 input을 올바르게 연결할 수 있는가?
- [ ] 유효성 검사 속성을 사용할 수 있는가?
- [ ] 체크박스와 라디오 버튼의 차이를 이해했는가?
- [ ] select와 textarea를 사용할 수 있는가?

### 미디어
- [ ] audio, video 태그를 사용할 수 있는가?
- [ ] iframe을 사용할 수 있는가?
- [ ] YouTube나 Google Maps를 임베드할 수 있는가?
- [ ] 미디어 접근성을 고려할 수 있는가?

### 마이크로데이터/ARIA
- [ ] 마이크로데이터의 기본 구조를 이해했는가?
- [ ] 기본적인 ARIA 속성을 사용할 수 있는가?
- [ ] ARIA 사용 가이드라인을 이해했는가?

## 참고 사항

- 모든 예제는 HTML5 표준을 따릅니다
- 접근성(accessibility)을 최우선으로 고려합니다
- 실제 웹사이트에서 사용할 수 있는 실용적인 예제입니다
- 일부 미디어 예제는 실제 파일이 없어 재생되지 않을 수 있습니다 (코드 구조 학습용)
