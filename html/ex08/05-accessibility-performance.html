메인 콘텐츠로 건너뛰기

접근성과 성능 예제

1. 색상 대비 (Color Contrast)

텍스트와 배경의 대비가 충분해야 합니다.

✅ 좋은 예 (충분한 대비)

이 텍스트는 배경과 충분한 대비를 가지고 있습니다.

색상: #333333, 배경: #ffffff (대비 비율: 12.6:1)

❌ 나쁜 예 (부족한 대비)

이 텍스트는 배경과 대비가 부족합니다.

색상: #cccccc, 배경: #ffffff (대비 비율: 1.6:1)

/* 좋은 예 */ .text { color: #333; /* 어두운 색 */ background-color: #fff; /* 밝은 배경 */ } /* 나쁜 예 */ .text { color: #ccc; /* 밝은 색 */ background-color: #fff; /* 밝은 배경 */ }
WCAG 기준:
  • 일반 텍스트: 최소 4.5:1
  • 큰 텍스트 (18px 이상): 최소 3:1

2. 포커스 관리

키보드 사용자를 위한 포커스 스타일이 필요합니다.

✅ 좋은 예 (명확한 포커스)

❌ 나쁜 예 (포커스 없음)

/* 좋은 예 */ .button:focus { outline: 3px solid #3498db; outline-offset: 2px; box-shadow: 0 0 0 3px rgba(52, 152, 219, 0.3); } /* 나쁜 예 */ .button:focus { outline: none; /* 접근성 문제 */ } /* 대안: 명확한 포커스 */ .button:focus { outline: none; box-shadow: 0 0 0 3px rgba(52, 152, 219, 0.5); }
주의: outline: none을 사용할 때는 반드시 대체 포커스 스타일을 제공해야 합니다.

3. 스킵 링크

키보드 사용자가 메인 콘텐츠로 바로 이동할 수 있는 링크입니다.

사용 방법:

  1. 페이지 상단에 스킵 링크 배치
  2. Tab 키를 누르면 첫 번째로 포커스
  3. Enter 키를 누르면 메인 콘텐츠로 이동
<a href="#main-content" class="skip-link"> 메인 콘텐츠로 건너뛰기 </a> .skip-link { position: absolute; top: -40px; /* 화면 밖 */ left: 0; background: #000; color: #fff; padding: 8px; } .skip-link:focus { top: 0; /* 포커스 시 보임 */ }

4. 성능: GPU 가속 활용

transform과 opacity는 GPU 가속을 받아 성능이 좋습니다.

✅ 좋은 예 (GPU 가속)

마우스를 올려보세요 (transform 사용)
/* 성능 좋음: GPU 가속 */ .element { transition: transform 0.3s; will-change: transform; } .element:hover { transform: translateY(-5px); /* GPU 가속 */ }

❌ 나쁜 예 (리플로우 발생)

마우스를 올려보세요 (left, width 사용)
/* 성능 나쁨: 리플로우 발생 */ .element { transition: left 0.3s, width 0.3s; } .element:hover { left: 20px; /* 리플로우 */ width: 300px; /* 리플로우 */ }
성능 좋은 속성:
  • transform (이동, 회전, 크기 조절)
  • opacity (투명도)
  • filter (필터 효과)
성능 나쁜 속성:
  • width, height (크기 변경)
  • top, left, right, bottom (위치 변경)
  • margin, padding (여백 변경)

5. will-change 사용

브라우저에 변경될 속성을 미리 알려줍니다.

.element { will-change: transform; /* transform이 변경될 예정 */ } .element:hover { transform: scale(1.2); }
주의사항:
  • 필요한 경우에만 사용
  • 사용 후 제거 권장
  • 과도한 사용은 오히려 성능 저하

6. 접근성 체크리스트

✅ 접근성 체크리스트

  • [ ] 색상 대비가 충분한가? (최소 4.5:1)
  • [ ] 포커스 스타일이 명확한가?
  • [ ] 키보드만으로 모든 기능 사용 가능한가?
  • [ ] 스크린 리더가 이해할 수 있는가?
  • [ ] 텍스트 크기 조절이 가능한가?
  • [ ] 색상만으로 정보를 전달하지 않는가?
  • [ ] 스킵 링크가 있는가?

7. 성능 최적화 팁

성능 최적화 팁

  • transform 사용: left/top 대신 transform 사용
  • opacity 사용: display: none 대신 opacity 사용 (가능한 경우)
  • will-change 신중 사용: 필요한 경우에만
  • 불필요한 애니메이션 제거: 성능에 영향 주는 애니메이션 최소화
  • CSS 최적화: 미사용 CSS 제거, 압축
  • Critical CSS: 중요한 CSS 인라인

8. 실전 예제: 접근성과 성능을 고려한 버튼

접근성과 성능을 모두 고려한 버튼 컴포넌트입니다.

.button { /* 성능: transform 사용 (GPU 가속) */ transition: transform 0.2s, box-shadow 0.2s; } .button:hover { transform: translateY(-2px); /* 성능 좋음 */ } /* 접근성: 명확한 포커스 */ .button:focus { outline: 3px solid #667eea; outline-offset: 2px; }
포함된 최적화:
  • ✅ transform 사용 (GPU 가속)
  • ✅ 명확한 포커스 스타일
  • ✅ 충분한 색상 대비
  • ✅ 적절한 크기 (터치 친화적)
