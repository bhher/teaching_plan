BEM 네이밍 전략 예제

1. BEM 구조 이해

BEM은 Block__Element--Modifier 구조를 따릅니다.

Block__Element--Modifier /* 예시 */ .card__title--large .button--primary .nav__link--active
구조 설명:
  • Block: 독립적인 컴포넌트 (예: .card, .button)
  • Element: Block의 일부 (예: .card__title, .button__icon)
  • Modifier: Block 또는 Element의 변형 (예: .card--large, .button--primary)

2. Card 컴포넌트 (BEM 적용)

BEM 방법론을 적용한 카드 컴포넌트입니다.

카드 제목

카드 부제목

이것은 BEM 방법론을 사용한 카드 컴포넌트입니다.

Block은 .card, Element는 .card__header, .card__title 등입니다.

<div class="card"> <div class="card__header"> <h3 class="card__title">카드 제목</h3> <p class="card__subtitle">카드 부제목</p> </div> <div class="card__body"> <p class="card__text">내용</p> </div> <div class="card__footer"> <button class="card__button card__button--primary">Primary</button> </div> </div> /* Block */ .card { } /* Element */ .card__header { } .card__title { } .card__body { } .card__text { } .card__footer { } .card__button { } /* Modifier */ .card__button--primary { } .card__button--secondary { }

3. Modifier 사용 예제

Modifier로 같은 Block의 변형을 만듭니다.

일반 카드

기본 카드 스타일입니다.

강조된 카드

card--highlighted Modifier가 적용되었습니다.

큰 카드

/* Block */ .card { } /* Modifier */ .card--highlighted { border: 3px solid #3498db; background-color: #e3f2fd; } .card--large { padding: 40px; }

4. Button 컴포넌트 (BEM 적용)

BEM 방법론을 적용한 버튼 컴포넌트입니다.

<button class="button button--primary">Primary</button> <button class="button button--secondary">Secondary</button> <button class="button button--primary button--large">Large</button> <button class="button button--primary"> <span class="button__icon">✓</span> With Icon </button> /* Block */ .button { } /* Modifier */ .button--primary { } .button--secondary { } .button--large { } .button--small { } /* Element */ .button__icon { }

5. Nav 컴포넌트 (BEM 적용)

BEM 방법론을 적용한 네비게이션 컴포넌트입니다.

  • 홈
  • 소개
  • 서비스
  • 연락처
<nav class="nav"> <ul class="nav__list"> <li class="nav__item"> <a href="#" class="nav__link nav__link--active">홈</a> </li> </ul> </nav> /* Block */ .nav { } /* Element */ .nav__list { } .nav__item { } .nav__link { } /* Modifier */ .nav__link--active { }

6. BEM 장점

✅ BEM의 장점

  • 명확한 구조: 클래스 이름만 봐도 구조 파악 가능
  • 재사용성: 컴포넌트 단위로 재사용
  • 특이성 관리: 특이성이 낮아 덮어쓰기 쉬움
  • 유지보수: 수정이 필요한 부분을 쉽게 찾을 수 있음
  • 협업: 일관된 네이밍으로 협업 효율 향상

7. 나쁜 네이밍 vs 좋은 네이밍

❌ 나쁜 예

.red-box { } /* 색상에 의존 */ .big-text { } /* 크기에 의존 */ .left-side { } /* 위치에 의존 */ .header1 { } /* 의미 없음 */

문제점: 디자인 변경 시 클래스 이름도 변경해야 함

✅ 좋은 예

.card { } /* 의미 있는 이름 */ .card--highlighted { } /* Modifier 사용 */ .card__title { } /* Element 사용 */ .button--primary { } /* 목적 명확 */

장점: 디자인 변경 시에도 클래스 이름 유지

8. BEM 규칙 요약

  1. Block: 독립적인 컴포넌트
    • 의미 있는 이름 사용
    • 재사용 가능해야 함
  2. Element: Block의 일부
    • Block__Element 형식
    • __ (언더스코어 2개)로 연결
    • Block 없이 독립적으로 사용 불가
  3. Modifier: Block 또는 Element의 변형
    • Block--Modifier 또는 Block__Element--Modifier
    • -- (하이픈 2개)로 연결
    • Block 또는 Element와 함께 사용
