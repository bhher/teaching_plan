뷰포트와 모바일 우선 예제

1. 뷰포트 메타 태그

뷰포트 메타 태그는 반응형 웹 디자인의 핵심입니다.

현재 페이지의 뷰포트 설정

<meta name="viewport" content="width=device-width, initial-scale=1.0">

설명:

  • width=device-width: 뷰포트 너비를 기기의 화면 너비로 설정
  • initial-scale=1.0: 초기 확대/축소 비율을 100%로 설정

❌ 뷰포트 메타 태그 없을 때

  • 모바일에서 데스크톱 페이지가 축소되어 표시됨
  • 텍스트가 너무 작아서 읽기 어려움
  • 사용자가 수동으로 확대해야 함
  • 반응형 CSS가 제대로 작동하지 않음

✅ 뷰포트 메타 태그 있을 때

  • 모바일에서 올바른 크기로 표시됨
  • 텍스트가 읽기 쉬운 크기
  • 반응형 CSS가 정상 작동
  • 최적의 사용자 경험

추가 옵션

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes">
  • maximum-scale=5.0: 최대 확대 비율 (접근성을 위해 설정 권장)
  • user-scalable=yes: 사용자 확대/축소 허용 (접근성 중요)

2. 모바일 우선 설계 (Mobile First)

작은 화면부터 디자인을 시작하여 점진적으로 큰 화면에 맞춰 확장합니다.

모바일 우선 예제

이 박스는 모바일 우선으로 디자인되었습니다.

화면 크기를 조절해보세요:

  • 모바일 (기본): 노란색, 작은 패딩
  • 태블릿 (768px 이상): 파란색, 중간 패딩
  • 데스크톱 (1024px 이상): 초록색, 큰 패딩
/* 기본: 모바일 */ .mobile-first-box { padding: 10px; background-color: #fff3cd; } /* 태블릿 이상 */ @media (min-width: 768px) { .mobile-first-box { padding: 20px; background-color: #e3f2fd; } } /* 데스크톱 */ @media (min-width: 1024px) { .mobile-first-box { padding: 30px; background-color: #e8f5e9; } }
모바일 우선의 장점:
  • 모바일 사용자는 필요한 CSS만 로드 (성능 향상)
  • 핵심 기능에 집중 (사용자 경험 향상)
  • 작은 화면부터 시작하여 점진적 향상 (개발 효율)

3. 데스크톱 우선 vs 모바일 우선 비교

데스크톱 우선 (비권장)

이 박스는 데스크톱 우선으로 디자인되었습니다.

모바일에서 불필요한 CSS를 로드합니다.

/* 기본: 데스크톱 (불필요한 CSS 로드) */ .desktop-first-box { padding: 30px; background-color: #e8f5e9; } /* 태블릿 이하 */ @media (max-width: 1023px) { .desktop-first-box { padding: 20px; } } /* 모바일 */ @media (max-width: 767px) { .desktop-first-box { padding: 10px; } }
데스크톱 우선의 문제점:
  • 모바일에서 불필요한 CSS 로드 (성능 저하)
  • 복잡한 미디어 쿼리 (유지보수 어려움)
  • 모바일 사용자 경험 저하
권장: 항상 모바일 우선으로 디자인하세요!

4. 모바일 우선 개발 순서

  1. 모바일 디자인
    • 핵심 기능에 집중
    • 간결한 레이아웃
    • 터치 친화적 (최소 44x44px)
  2. 태블릿 확장
    • 추가 공간 활용
    • 2열 레이아웃
    • 더 많은 콘텐츠 표시
  3. 데스크톱 최적화
    • 최대 공간 활용
    • 복잡한 레이아웃
    • 호버 효과 등

5. 뷰포트 테스트

브라우저 개발자 도구를 사용하여 다양한 화면 크기를 테스트하세요.

테스트 방법:
  1. F12로 개발자 도구 열기
  2. 디바이스 모드 아이콘 클릭 (Ctrl+Shift+M)
  3. 다양한 기기 크기 선택
  4. 실제 기기에서도 테스트 권장
