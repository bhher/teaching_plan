## HTML 6차시 – 시맨틱 태그로 의미 있는 레이아웃 만들기

### 0. 학습 목표

- `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`의 역할을 이해한다.
- `div`와 시맨틱 태그의 차이를 설명할 수 있다.
- 시맨틱 태그가 **웹 접근성(스크린 리더, 검색 엔진)** 에 왜 중요한지 이해한다.
- 시맨틱 태그를 이용해 **기본 웹 페이지 레이아웃**을 직접 구성할 수 있다.

---

### 1. 시맨틱(semantic) 태그란?

#### 1) 의미 있는 태그

- 시맨틱 태그 = **“이 구역이 어떤 역할을 하는지” 이름만 보고도 알 수 있는 태그**
  - 예: `header` → 머리말, `nav` → 메뉴/네비게이션, `footer` → 바닥글
- 반대 개념: `div`
  - 단순한 “상자(Box)” → 역할은 CSS/문맥을 봐야만 알 수 있음

```html
<!-- 시맨틱하지 않은 예 -->
<div id="top"></div>
<div class="menu"></div>
<div id="content"></div>

<!-- 시맨틱 태그 사용 예 -->
<header></header>
<nav></nav>
<main></main>
```

> 비유:  
> 사람을 설명할 때 “이 사람은 회사에서 사장입니다” 라고 말해 주는 것과,  
> “이 사람은 그냥 사람입니다” 라고만 말해 주는 것의 차이.

---

### 2. 주요 시맨틱 태그 소개

#### 2.1 `header` – 머리말 영역

- 페이지나 섹션의 **상단 부분(머리)** 을 나타냄
- 보통 로고, 사이트 제목, 메인 메뉴 일부가 위치

```html
<header>
  <h1>나의 웹사이트</h1>
  <p>간단한 소개 문구</p>
</header>
```

#### 2.2 `nav` – 네비게이션(메뉴)

- 사이트 내부의 **주요 링크 모음**을 나타냄
  - 상단 메뉴, 사이드 메뉴, 푸터 메뉴 등

```html
<nav>
  <ul>
    <li><a href="index.html">홈</a></li>
    <li><a href="about.html">소개</a></li>
    <li><a href="contact.html">연락처</a></li>
  </ul>
</nav>
```

#### 2.3 `main` – 문서의 주요 내용

- 페이지에서 **가장 중요한 주요 콘텐츠**를 감싸는 태그
- 한 페이지에 **`<main>`은 하나만** 사용하는 것이 원칙

```html
<main>
  <h2>오늘의 글</h2>
  <p>여기에 본문 내용이 들어갑니다.</p>
</main>
```

#### 2.4 `section` – 주제별 구역

- 문서 안의 **주제별 구분 영역**
  - 예: “공지사항”, “인기 글”, “추천 글” 섹션 등

```html
<section>
  <h2>공지사항</h2>
  <ul>
    <li>공지 1</li>
    <li>공지 2</li>
  </ul>
</section>
```

#### 2.5 `article` – 독립된 글/콘텐츠

- **독립적으로 떼어내도 의미가 있는 콘텐츠** 묶음
  - 예: 블로그 글, 뉴스 기사, 게시판 글 하나

```html
<article>
  <h3>블로그 글 제목</h3>
  <p>글 내용 일부...</p>
</article>
```

#### 2.6 `aside` – 보조 정보

- 본문(main)과 **직접적인 관련은 없지만, 참고용으로 볼 수 있는 내용**
  - 예: 광고, 인기 글 목록, 관련 링크, 프로필 박스 등

```html
<aside>
  <h3>관련 링크</h3>
  <ul>
    <li><a href="#">이전 글</a></li>
    <li><a href="#">다음 글</a></li>
  </ul>
</aside>
```

#### 2.7 `footer` – 바닥글

- 페이지나 섹션의 **하단 영역**
  - 예: 저작권 문구, 연락처, 회사 정보, 저자 정보 등

```html
<footer>
  <p>© 2025 My Website. All rights reserved.</p>
</footer>
```

### 3. div와 시맨틱 태그의 차이

#### 3.1 `div`는 의미 없는 상자

- `div` 자체로는 “이게 무슨 역할인지” 알 수 없음
- CSS로 레이아웃을 잡거나, JS로 묶어서 조작할 때 유용한 **범용 박스**

```html
<div class="header"></div>   <!-- 이걸 보면 직관적으로 header인지 알기 어려움 -->
```

#### 3.2 시맨틱 태그는 이름만으로 역할이 드러남

```html
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
```

- 코드만 봐도 대략적인 문서 구조를 이해할 수 있음

> 정리:  
> - **div**: “이름 없는 상자” → 스타일링/레이아웃에 많이 사용  
> - **시맨틱 태그**: “이름 있는 상자” → 구조와 의미 전달에 유리

실무에서는:
- 큰 구조는 `header`, `main`, `footer`, `section`, `article` 등을 사용하고  
- 그 안에서 세부 레이아웃을 `div`로 잡는 경우가 많다.

---

### 4. 웹 접근성 관점에서의 시맨틱 태그

#### 4.1 스크린 리더와 “랜드마크(landmark)”

- 시맨틱 태그는 **스크린 리더가 문서 구조를 이해하는 데 도움**을 준다.
  - 예: 스크린 리더가 “여기는 main 영역입니다”, “여기는 navigation입니다” 라고 안내
- 사용자는 키보드만으로도 **헤더/메인/푸터를 빠르게 이동**할 수 있다.

#### 4.2 검색 엔진(SEO)에도 유리

- 검색 엔진은 문서의 구조를 해석해 “어떤 부분이 중요한 내용인지” 파악
  - 예: `article` 안의 텍스트를 “본문 내용”으로 인식

> 요약:  
> 시맨틱 태그는 “사람에게도 읽기 쉽고, 컴퓨터에게도 이해하기 쉬운 HTML”을 만드는 도구이다.

---

### 5. 구조 그림을 글로 비유해서 설명하기

**예시 레이아웃(블로그/뉴스 사이트 형태)을 말로 묘사:**

1. 맨 위에 **`header`** 가 있고,  
   그 안에는 사이트 제목과 간단한 소개 문구가 있다.
2. 그 아래에 **`nav`** 가 있어,  
   “홈 / 블로그 / 소개” 메뉴 링크들이 가로로 나열되어 있다.
3. 화면 가운데는 **`main`** 이 가장 넓은 영역을 차지하고,  
   그 안에는 여러 개의 **`article`** 이 세로로 쌓여 있다.  
   각 article은 “글 제목(h2) + 글 내용(p)”으로 구성된다.
4. main의 오른쪽에는 조금 좁은 칸으로 **`aside`** 가 있다.  
   여기에는 “인기 글 목록, 광고, 프로필 박스” 같은 보조 정보가 들어간다.
5. 화면 맨 아래에는 **`footer`** 가 있고,  
   그 안에 저작권 문구와 연락처 정보가 적혀 있다.

글로 표현하면:

```txt
[HEADER]
  사이트 제목, 소개 문구

[NAV]
  홈 | 블로그 | 소개

[MAIN]---------------------[ASIDE]
  [ARTICLE 1]               인기 글
    제목 + 글내용          광고
  [ARTICLE 2]               ...
    제목 + 글내용
  ...

[FOOTER]
  © 2025 사이트 이름, 연락처
```

이 구조를 코드로 옮길 때 시맨틱 태그를 사용하면,  
코드만 봐도 위 그림이 자연스럽게 떠오르도록 하는 것이 목표다.

### 6. 실습 – 기본 웹 페이지 레이아웃 만들기

#### 6.1 목표

- `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`를 모두 사용해  
  **간단한 웹 페이지 골격**을 만든다.

#### 6.2 예시 코드 스켈레톤

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>시맨틱 태그 레이아웃 예제</title>
  </head>
  <body>
    <header>
      <h1>나의 웹사이트</h1>
      <p>시맨틱 태그 연습 페이지입니다.</p>
    </header>

    <nav>
      <ul>
        <li><a href="#">홈</a></li>
        <li><a href="#">블로그</a></li>
        <li><a href="#">소개</a></li>
      </ul>
    </nav>

    <main>
      <section>
        <h2>최근 글</h2>

        <article>
          <h3>첫 번째 글 제목</h3>
          <p>첫 번째 글의 내용 일부를 여기에 작성합니다.</p>
        </article>

        <article>
          <h3>두 번째 글 제목</h3>
          <p>두 번째 글의 내용 일부를 여기에 작성합니다.</p>
        </article>
      </section>

      <aside>
        <h2>사이드바</h2>
        <p>여기는 보조 정보 영역입니다.</p>
        <ul>
          <li><a href="#">인기 글 1</a></li>
          <li><a href="#">인기 글 2</a></li>
        </ul>
      </aside>
    </main>

    <footer>
      <p>© 2025 나의 웹사이트</p>
      <p>이메일: example@example.com</p>
    </footer>
  </body>
</html>
```

> 수업에서는 위 코드를 **완성본 예시**로 보여주고,  
> 학생들에게는 텍스트 내용(제목, 문장, 링크 텍스트)을 각자 바꿔보게 합니다.

### 7. 실습 응용 아이디어 (강사용)

- “블로그 레이아웃”, “뉴스 사이트 레이아웃”, “쇼핑몰 레이아웃” 중 하나를 골라  
  **시맨틱 태그 이름만 써서 구조를 먼저 설계**하게 한 뒤,  
  나중에 실제 HTML로 옮겨 쓰게 한다.

예:

```txt
쇼핑몰 예시:

HEADER: 로고, 검색창
NAV: 카테고리 메뉴
MAIN:
  SECTION: 오늘의 할인 상품
    ARTICLE: 상품 카드들
  SECTION: 추천 상품
    ARTICLE: 상품 카드들
ASIDE: 장바구니 미리보기, 광고
FOOTER: 회사 정보, 고객센터
```

---

### 8. 마무리 정리

- 시맨틱 태그:
  - `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`
  - 문서의 구조와 의미를 명확하게 표현하는 도구
- `div`와의 차이:
  - `div`는 단순 상자, 시맨틱 태그는 역할이 드러나는 상자
- 웹 접근성:
  - 스크린 리더와 검색엔진이 문서 구조를 잘 이해하게 도와줌
- 실습:
  - 시맨틱 태그를 활용해 기본 레이아웃을 만들며,  
    “사람에게도, 컴퓨터에게도 친절한 HTML”을 경험해 본다.

> 다음 차시 예고:  
> “다음 시간에는 지금 만든 시맨틱 레이아웃에 CSS를 입혀서  
> 실제 서비스에 가까운 형태로 꾸며 보겠습니다.”
