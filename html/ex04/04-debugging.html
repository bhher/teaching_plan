레이아웃 디버깅 기초

1. 개발자 도구 사용법

브라우저의 개발자 도구는 레이아웃 문제를 해결하는 가장 강력한 도구입니다.

단축키

  • F12: 개발자 도구 열기/닫기
  • Ctrl+Shift+C (Windows) / Cmd+Option+C (Mac): 요소 선택 도구
  • Ctrl+Shift+I (Windows) / Cmd+Option+I (Mac): 개발자 도구 열기

주요 탭

  • Elements: HTML 구조와 CSS 스타일 확인
  • Computed: 최종 적용된 스타일과 박스 모델 시각화
  • Styles: 적용된 CSS 규칙 확인
  • Console: JavaScript 에러 확인
  • Network: 리소스 로딩 확인
팁: Elements 탭에서 요소를 선택하면 Computed 탭에서 박스 모델을 시각적으로 확인할 수 있습니다.

2. 문제 1: 요소가 보이지 않음

원인 1: display: none

이 요소는 display: none으로 숨겨져 있습니다.

위의 요소는 공간도 차지하지 않습니다.

.hidden { display: none; /* 완전히 숨김 */ }

원인 2: width: 0 또는 height: 0

이 요소는 크기가 0입니다.
.zero-size { width: 0; height: 0; overflow: hidden; }

원인 3: opacity: 0

이 요소는 opacity: 0으로 투명합니다. (공간은 차지함)

해결 방법

  1. 개발자 도구에서 요소 선택
  2. Computed 탭에서 display, width, height 확인
  3. Styles 탭에서 관련 속성 확인

3. 문제 2: 레이아웃이 깨짐

원인 1: Box-sizing 문제

content-box: 실제 너비는 250px
border-box: 실제 너비는 200px

원인 2: Float 문제

Float 요소

부모의 높이가 사라졌습니다.

Float 요소

Clearfix로 해결했습니다.

/* Clearfix */ .clearfix::after { content: ""; display: table; clear: both; }

원인 3: Margin Collapse

margin-bottom: 30px
margin-top: 20px
실제 간격: 30px (큰 값)

4. 문제 3: 스타일이 적용되지 않음

원인 1: 특이성(Specificity) 문제

인라인 스타일이 적용됨 (특이성: 1000점)

/* 특이성: 1점 - 적용 안 됨 */ p { color: blue; } /* 특이성: 10점 - 적용 안 됨 */ .test-class { color: green; } /* 특이성: 100점 - 적용 안 됨 */ #test-id { color: red; } /* 특이성: 1000점 - 적용됨 */ style="color: purple;"

원인 2: 잘못된 선택자

주의: 선택자가 올바른 요소를 선택하는지 확인하세요.

해결 방법

  1. Styles 탭에서 비활성화된 스타일 확인 (취소선)
  2. 특이성 계산
  3. 브라우저 캐시 삭제 (Ctrl+Shift+R)

5. 디버깅 체크리스트

✅ 체크리스트

  • [ ] 개발자 도구로 요소 선택
  • [ ] Computed 탭에서 박스 모델 확인
  • [ ] Styles 탭에서 적용된 규칙 확인
  • [ ] 비활성화된 스타일 확인 (취소선)
  • [ ] Console 탭에서 에러 확인
  • [ ] Network 탭에서 리소스 로딩 확인
  • [ ] Box-sizing 확인
  • [ ] Float clear 확인
  • [ ] Margin collapse 확인

6. 유용한 디버깅 기법

기법 1: 임시 테두리 추가

모든 요소에 테두리를 추가하여 레이아웃을 시각화합니다.
/* 개발자 도구 Console에서 실행 */ document.querySelectorAll('*').forEach(el => { el.style.border = '1px solid red'; });

기법 2: 배경색 추가

배경색을 추가하여 요소의 영역을 확인합니다.

기법 3: 콘솔에서 요소 확인

// 개발자 도구 Console에서 $0 // 선택된 요소 $0.offsetWidth // 요소 너비 $0.offsetHeight // 요소 높이 $0.offsetTop // 상단 위치 $0.offsetLeft // 왼쪽 위치 getComputedStyle($0) // 계산된 스타일

기법 4: 박스 모델 계산기

팁: 개발자 도구의 Computed 탭에서 자동으로 계산된 값을 확인할 수 있습니다. Box Model 섹션에서 margin, border, padding, content를 시각적으로 확인하세요.
