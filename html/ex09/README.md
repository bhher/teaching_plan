## 사용 방법

1. 각 예제 파일을 참고하여 프로젝트 구조를 설계하세요
2. package.json을 복사하여 프로젝트에 맞게 수정하세요
3. .gitignore를 프로젝트에 추가하세요
4. 스타일 가이드를 참고하여 코딩 규칙을 정하세요
5. Git 워크플로우를 따라 협업하세요

## 프로젝트 구조 선택 가이드

### 작은 프로젝트 (1-5페이지)
- 기본 구조 사용
- 파일 수가 적어 관리 쉬움
- 빠른 개발 가능

### 중간 프로젝트 (5-20페이지)
- 확장 구조 사용
- HTML과 자산 파일 분리
- CSS를 기능별로 분리

### 큰 프로젝트 (20페이지 이상)
- 세밀한 구조 사용
- 컴포넌트 기반 설계
- 빌드 도구 필요

## npm Scripts 사용법

### 개발 서버 실행
```bash
npm run dev
```

### 빌드
```bash
npm run build
```

### CSS 빌드
```bash
npm run build:css
```

### JavaScript 빌드
```bash
npm run build:js
```

### 파일 감시
```bash
npm run watch
```

## Git 워크플로우

### 기본 워크플로우
1. 기능 브랜치 생성
2. 개발 및 커밋
3. 원격 저장소에 푸시
4. Pull Request 생성
5. 코드 리뷰 후 병합

### 브랜치 네이밍
- `feature/`: 새로운 기능
- `fix/`: 버그 수정
- `hotfix/`: 긴급 수정
- `refactor/`: 리팩토링

### 커밋 메시지 형식
```
타입(범위): 간단한 설명

상세 설명 (선택사항)
```

## 스타일 가이드

### HTML 규칙
- 들여쓰기: 2칸
- 속성 따옴표: 쌍따옴표
- 시맨틱 태그 사용

### CSS 규칙
- 네이밍: BEM 방법론
- 속성 순서: 위치 → 박스 → 시각 → 타이포 → 기타
- 들여쓰기: 2칸

### JavaScript 규칙
- 변수: const 우선
- 함수: 화살표 함수 또는 함수 선언
- 네이밍: camelCase

## 리팩토링 가이드

### 리팩토링이 필요한 경우
- 중복 코드 발견
- 긴 함수/클래스
- 복잡한 조건문
- 매직 넘버/문자열
- 명확하지 않은 변수명

### 리팩토링 원칙
1. 작은 단계로 진행
2. 기능 유지
3. 테스트
4. 일관성
