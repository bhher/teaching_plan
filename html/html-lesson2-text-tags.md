## HTML 2차시 – 텍스트 태그 기초 (제목, 문단, 강조)

### 0. 학습 목표

- HTML에서 **제목(h1~h6)**, **문단(p)**, **줄바꿈(br)**, **구분선(hr)**, **강조(strong/em)** 태그를 사용할 수 있다.
- 제목 태그를 **문서 구조(목차)** 관점에서 이해한다.
- 자기소개 페이지를 직접 작성해 본다.
- 자주 하는 실수(h1 중복, p 안에 블록 태그 넣기)를 피할 수 있다.

---

### 1. 제목 태그 – `<h1>` ~ `<h6>`

#### 1) 개념

- `<h1>` ~ `<h6>`: **Heading (제목)** 태그
  - `<h1>`: 가장 큰 제목 (문서 전체의 주제)
  - `<h2>`: `<h1>` 아래의 소제목
  - `<h3>`: `<h2>` 아래의 작은 제목 … 이런 식으로 계층 구조

```html
<h1>나의 블로그</h1>
<h2>오늘의 일기</h2>
<h3>아침</h3>
<h3>점심</h3>
<h3>저녁</h3>
<h2>내일의 계획</h2>
```

#### 2) 신체/책 비유

- HTML 문서 = **책 한 권**
  - `<h1>` = 책 제목
  - `<h2>` = 큰 목차(챕터 제목)
  - `<h3>` = 챕터 안의 소제목

> **중요**: 일반적으로 한 페이지에 **`<h1>`은 하나만** 사용하는 것을 권장  
> (`<h1>`이 책 제목이 여러 개면 헷갈리듯이, 문서의 최상위 주제는 하나인 경우가 많기 때문)

---

### 2. 문단 태그 – `<p>`

#### 1) 개념

- `<p>`: paragraph, **문단 하나**를 의미
- 자동으로 위아래에 약간의 여백(마진)이 생김

```html
<p>안녕하세요. 저는 HTML을 공부 중입니다.</p>
<p>문단은 이렇게 p 태그로 나누어 줍니다.</p>
```

#### 2) 자주 하는 실수 – `<p>` 안에 다른 블록 태그 넣기

```html
<!-- ❌ 안 좋은 예 -->
<p>
  이건 문단입니다.
  <h2>이건 제목인데 p 안에 들어있어요.</h2>
</p>
```

- `<p>` 안에는 **다른 블록 요소(h1, h2, div, ul 등)**를 넣지 않는 것이 원칙
- `<p>`는 “문단 하나”이기 때문에, 제목/리스트/카드 등은 **p 밖에 따로** 작성

```html
<!-- ✅ 올바른 예 -->
<p>이건 문단입니다.</p>
<h2>이건 제목입니다.</h2>
```

---

### 3. 줄바꿈 / 구분선 – `<br>`, `<hr>`

#### 3.1 줄바꿈 – `<br>`

- `<br>`: line break, **강제 줄바꿈**
- 문단 안에서 줄을 바꾸고 싶을 때 사용

```html
<p>
안녕하세요.<br />
저는 홍길동입니다.<br />
HTML 공부가 재미있습니다.
</p>
```

> 주의: 문단 자체를 나누고 싶다면 `<p>`를 여러 개 쓰는 것이 더 좋다.  
> `<br>`는 “같은 문단 안에서 줄 바꿈” 용도로만 사용하는 습관을 들이자.

#### 3.2 구분선 – `<hr>`

- `<hr>`: horizontal rule, **가로줄(구분선)** 을 그어준다.

```html
<h1>나의 자기소개</h1>
<p>안녕하세요. 저는 코딩을 처음 배우는 학생입니다.</p>

<hr />

<h2>취미</h2>
<p>독서, 걷기, 음악 듣기를 좋아합니다.</p>
```

---

### 4. 강조 태그 – `<strong>`, `<em>`

#### 4.1 `<strong>` – 강한 강조 (굵게 + 의미)

- 기본 스타일: **굵게(bold)**
- 의미: “이 단어/문장이 **특히 중요**하다”

```html
<p>오늘 수업에서 가장 중요한 태그는 <strong>h1</strong> 입니다.</p>
```

#### 4.2 `<em>` – 강조 (기울임 + 의미)

- 기본 스타일: *기울임(italic)*
- 의미: 말할 때 **강조해서 읽는 부분**

```html
<p>이 부분은 <em>정말</em> 주의해서 봐야 합니다.</p>
```

#### 4.3 시각적인 스타일 vs 의미

- `<b>`, `<i>` 태그도 있지만,  
  `<strong>`, `<em>`은 “이 내용이 **중요하다**”는 의미까지 함께 전달
  - 스크린 리더(시각장애인용 리더기) 등에서도 더 의미 있게 인식

> 실무 팁:  
> 단순히 진하게만 보이고 싶을 때는 CSS로 `font-weight`를 변경하고,  
> 정말 강조하고 싶은 단어에는 `<strong>`, `<em>`을 사용하면 좋다.

### 5. 실습 – 자기소개 페이지 만들기

#### 5.1 목표

- 지금까지 배운 텍스트 태그들을 이용해서  
  **자기소개 페이지 (`intro.html` 등)** 를 만든다.

#### 5.2 요구사항

- 필수로 사용할 태그:
  - `<h1>`: 페이지의 가장 큰 제목 (자기 이름 등)
  - `<h2>` 또는 `<h3>`: “소개”, “취미”, “관심사” 등의 소제목
  - `<p>`: 각 항목의 설명 문단
  - `<br>`: 한 문단 안에서 줄을 바꾸고 싶을 때 1~2번 사용
  - `<hr>`: 큰 구역을 나누는 구분선 1개 이상
  - `<strong>`, `<em>`: 중요한 단어나 문장을 강조

#### 5.3 예시 구조(코드 예시는 수업에서 함께 작성)

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>홍길동 자기소개</title>
  </head>
  <body>
    <h1>홍길동 자기소개</h1>

    <p>
      안녕하세요. 저는 <strong>홍길동</strong>입니다.<br />
      코딩을 <em>처음</em> 배우고 있습니다.
    </p>

    <hr />

    <h2>취미</h2>
    <p>독서, 산책, 음악 감상</p>

    <h2>관심사</h2>
    <p>웹 개발, 디자인, 사용자 경험(UX)에 관심이 있습니다.</p>
  </body>
</html>
```

> 실제 수업에서는 위 코드 전체를 보여주기보다는,  
> **빈 틀**만 제공하고 학생들이 직접 내용을 채워 넣게 하는 것이 좋습니다.

### 6. 자주 하는 실수와 피하는 방법

#### 6.1 `h1`을 여러 번 쓰는 경우

```html
<!-- ❌ 안 좋은 예 -->
<h1>나의 블로그</h1>
<h1>오늘의 일기</h1>
<h1>내일의 계획</h1>
```

- 문제:
  - 문서의 최상위 제목이 여러 개라서 **구조가 불분명**
  - 검색 엔진, 스크린 리더 등에서 문서 구조를 파악하기 어려움

**권장 방법:**

- 문서당 `h1`은 **보통 1개**만 사용하고, 그 아래는 `h2`, `h3`로 단계 내려가기

```html
<!-- ✅ 권장 예 -->
<h1>나의 블로그</h1>

<h2>오늘의 일기</h2>
<p>...</p>

<h2>내일의 계획</h2>
<p>...</p>
```

---

#### 6.2 `<p>` 안에 다른 블록 태그 넣기

```html
<!-- ❌ 안 좋은 예 -->
<p>
  안녕하세요.
  <h2>취미</h2>
  저는 독서를 좋아합니다.
</p>
```

- `<p>` 태그는 **문단 하나**를 감싸기 위한 태그
- `<h2>`, `<div>`, `<ul>` 같은 블록 요소는 `<p>` 밖에서 독립적으로 사용

```html
<!-- ✅ 올바른 예 -->
<p>안녕하세요.</p>

<h2>취미</h2>
<p>저는 독서를 좋아합니다.</p>
```

> 지도 팁:  
> 학생들이 `<p>` 안에 제목이나 리스트를 넣으면,  
> “이 부분은 **문단**인가요, 아니면 **제목/리스트**인가요?”라고 물어보고  
> 역할에 따라 태그를 분리하도록 유도합니다.

---

### 7. 마무리 정리

- **제목 태그(h1~h6)** 는 문서의 **구조(목차)** 를 만드는 역할
- **문단(p)** 은 글의 단락을 나타냄
- **강제 줄바꿈(br)**, **구분선(hr)** 으로 문장을 더 보기 좋게 정리
- **강조(strong/em)** 로 중요한 부분을 눈에 띄게 표시
- 실습: 자기소개 페이지를 직접 만들어 보며, 태그들의 역할을 몸으로 익힌다.

> 다음 차시 예고:  
> “다음 시간에는 텍스트 외에도 **링크(a 태그)**, **이미지(img 태그)** 를 사용해서  
> 실제 웹 페이지처럼 다른 사이트로 이동하고, 사진을 보여주는 페이지를 만들어 보겠습니다.”
