## HTML 3차시 – 목록 & 링크 태그

### 0. 학습 목표

- **순서 있는 목록(ol)**, **순서 없는 목록(ul)**, **항목(li)** 태그를 사용할 수 있다.
- 링크를 만드는 **`<a>` 태그와 `href` 속성**의 역할을 이해한다.
- 링크 열기 방식인 **`_blank`와 `_self`**의 차이를 설명할 수 있다.
- 메뉴 목록과 외부 사이트 링크를 포함한 간단한 페이지를 직접 만든다.

---

### 1. 목록 태그 – `<ul>`, `<ol>`, `<li>`

#### 1) `<ul>` – 순서 없는 목록 (unordered list)

- 앞에 동그라미(불릿)가 붙는 목록
- “순서가 중요하지 않은 항목들”을 나열할 때 사용

```html
<h2>좋아하는 음식</h2>
<ul>
  <li>피자</li>
  <li>치킨</li>
  <li>라면</li>
</ul>
```

#### 2) `<ol>` – 순서 있는 목록 (ordered list)

- 1, 2, 3... 번호가 붙는 목록
- 순서가 중요한 항목(레시피 단계, 순위 등)에 사용

```html
<h2>아침 준비 순서</h2>
<ol>
  <li>일어나기</li>
  <li>세수하기</li>
  <li>아침 먹기</li>
</ol>
```

#### 3) `<li>` – 목록 항목 (list item)

- `<ul>` 또는 `<ol>` 안에서만 사용
- 각각의 한 줄 항목을 의미

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

> 팁:  
> `<ul>`/`<ol>` 안에는 기본적으로 **`<li>`만 직접 자식으로** 두는 것을 권장합니다.  
> (필요하면 `<li>` 안에서 `<p>`나 `<a>` 등을 사용)

---

### 2. 링크 태그 – `<a>`와 `href`

#### 2.1 `<a>` 태그 기본

- `<a>` : anchor (닻) 태그, **링크**를 만드는 태그
- **어디로 이동할지**는 `href` 속성으로 지정

```html
<a href="https://www.google.com">구글로 이동</a>
```

- 사용자가 “구글로 이동” 글자를 클릭하면,  
  `https://www.google.com` 주소로 이동

#### 2.2 `href` 속성

- **Hypertext REFerence** 의 약자
- 이동할 **목표 주소(URL)** 를 적는다.

예시:

```html
<!-- 외부 사이트로 이동 -->
<a href="https://www.naver.com">네이버</a>

<!-- 같은 사이트 안의 다른 페이지로 이동 -->
<a href="about.html">소개 페이지</a>
```

---

### 3. 링크 열기 방식 – `_self`, `_blank`

#### 3.1 `target` 속성

- 링크를 클릭했을 때 **어떻게** 열릴지 정하는 옵션
- 자주 쓰는 값:
  - `_self` : **현재 탭(창)** 에서 열기 (기본값)
  - `_blank`: **새 탭**에서 열기

```html
<!-- 현재 탭에서 열기 (target 생략 시 기본값) -->
<a href="https://www.naver.com" target="_self">네이버 (현재 탭)</a>

<!-- 새 탭에서 열기 -->
<a href="https://www.google.com" target="_blank">구글 (새 탭)</a>
```

> 사용자 경험 관점:  
> - 내 사이트 안의 다른 페이지로 이동 → 보통 `_self`  
> - 외부 사이트(참고 자료 등)를 보여줄 때 → `_blank` 를 쓰는 경우가 많다.

---

### 4. 실습 1 – 메뉴 목록 만들기

#### 4.1 목표

- `<ul>` + `<li>` + `<a>`를 이용해 **간단한 메뉴**를 만든다.

#### 4.2 예시 구조

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>메뉴 예제</title>
  </head>
  <body>
    <h1>나의 홈페이지</h1>

    <h2>메뉴</h2>
    <ul>
      <li><a href="index.html">홈</a></li>
      <li><a href="about.html">소개</a></li>
      <li><a href="contact.html">연락처</a></li>
    </ul>
  </body>
</html>
```

> 수업에서는 위 구조를 함께 만들되,  
> 학생들에게 메뉴 이름과 링크 주소를 **스스로 정해 보게** 하는 것이 좋습니다.

---

### 5. 실습 2 – 외부 사이트 링크 연결

#### 5.1 목표

- 메뉴 또는 별도 섹션에서 **외부 사이트로 연결되는 링크**를 만든다.
- `_blank`와 `_self` 차이를 직접 체험해 본다.

#### 5.2 실습 예시

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>링크 실습</title>
  </head>
  <body>
    <h1>즐겨찾는 사이트</h1>

    <h2>국내 사이트</h2>
    <ul>
      <li>
        <a href="https://www.naver.com" target="_self">네이버 (현재 탭)</a>
      </li>
      <li>
        <a href="https://www.daum.net" target="_blank">다음 (새 탭)</a>
      </li>
    </ul>

    <h2>개발 관련 사이트</h2>
    <ul>
      <li>
        <a href="https://developer.mozilla.org/ko/" target="_blank">
          MDN 웹 문서
        </a>
      </li>
      <li>
        <a href="https://stackoverflow.com" target="_blank">
          Stack Overflow
        </a>
      </li>
    </ul>
  </body>
</html>
```

학생들에게는:
- **본인이 자주 사용하는 사이트 3~5개**를 각자 적어보게 하고,
- 어떤 링크는 `_self`, 어떤 링크는 `_blank`로 설정해 보게 한다.

---

### 6. 종합 실습 – 간단한 “포트폴리오 메뉴” 만들기

#### 6.1 목표

- 지금까지 배운 텍스트(tag 2차시) + 목록/링크(tag 3차시)를 합쳐서,
  **작은 포트폴리오 메뉴 페이지**를 만든다.

#### 6.2 요구사항

- 페이지 제목: `<h1>` 사용 (예: “홍길동의 포트폴리오”)
- 섹션 2~3개:
  - 예: “소개”, “기술 스택”, “링크 모음”
- 각 섹션에:
  - `<h2>` 제목
  - `<p>` 간단 설명
  - `<ul>` + `<li>` 목록
  - `<a>` 태그로 외부/내부 링크 최소 2개 이상
- 외부 링크 중 1개는 `_blank`, 1개는 `_self` 사용

#### 6.3 교사용 체크 리스트

- `ul`/`ol` 안에 `li`만 있는지 확인
- `a` 태그에 `href`가 빠지지 않았는지 확인
- `_blank`와 `_self`를 각각 어디에 쓰는지 말로 설명할 수 있는지 질문

---

### 7. 마무리 정리

- `<ul>` / `<ol>` / `<li>` 로 **목록**을 만들 수 있다.
  - 순서가 없는 내용 → `<ul>`
  - 순서가 중요한 내용 → `<ol>`
- `<a>`와 `href` 로 **링크**를 만든다.
- `target="_self"` : 현재 탭에서 열기 (기본값)  
  `target="_blank"` : 새 탭에서 열기
- 실습: 메뉴 목록 + 외부 링크를 통해  
  실제 웹사이트처럼 페이지를 이동해 보는 경험을 한다.

> 다음 차시 예고:  
> “다음 시간에는 이미지(`<img>`)와 파일 경로 개념을 배우고,  
> 사진이 들어간 나만의 프로필 페이지를 만들어 보겠습니다.”
