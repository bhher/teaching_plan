# 자바 작업형 평가 문제 1 (추가 문제) - 정답

## 문제: Product 클래스 작성

### 정답 코드

```java
public class Product {
    String name;    // 상품명
    int price;     // 가격
    int stock;     // 재고 수량
    
    // 기본 생성자
    public Product() {
    }
    
    // 매개변수 생성자
    public Product(String name, int price, int stock) {
        this.name = name;
        this.price = price;
        this.stock = stock;
    }
    
    // toString() 메서드
    @Override
    public String toString() {
        return "상품명: " + name + ", 가격: " + price + "원, 재고: " + stock + "개";
    }
    
    // 총 가치 계산 메서드
    public int getTotalValue() {
        return price * stock;
    }
    
    // 재고 확인 메서드
    public boolean isAvailable() {
        return stock > 0;
    }
}
```

---

## 코드 설명

### 1. 필드 선언 (5점)
```java
String name;    // 상품명
int price;      // 가격
int stock;      // 재고 수량
```
- 클래스의 속성을 나타내는 변수들
- 각 필드는 객체마다 다른 값을 가질 수 있음

### 2. 기본 생성자 (2점)
```java
public Product() {
}
```
- 매개변수가 없는 생성자
- 객체를 기본값으로 초기화할 때 사용

### 3. 매개변수 생성자 (3점)
```java
public Product(String name, int price, int stock) {
    this.name = name;
    this.price = price;
    this.stock = stock;
}
```
- 필드를 초기화하는 생성자
- `this` 키워드로 필드와 매개변수를 구분

### 4. toString() 메서드 (5점)
```java
@Override
public String toString() {
    return "상품명: " + name + ", 가격: " + price + "원, 재고: " + stock + "개";
}
```
- 객체를 문자열로 표현하는 메서드
- `System.out.println()`에서 자동으로 호출됨
- `@Override` 어노테이션으로 부모 클래스 메서드 재정의를 명시

### 5. getTotalValue() 메서드 (3점)
```java
public int getTotalValue() {
    return price * stock;
}
```
- 가격과 재고를 곱하여 총 가치 계산
- 반환 타입이 int이므로 계산 결과를 그대로 반환

### 6. isAvailable() 메서드 (2점)
```java
public boolean isAvailable() {
    return stock > 0;
}
```
- 재고가 있는지 확인하는 메서드
- 조건식을 직접 반환 가능 (간결한 코드)

## 실행 결과 확인

### 실행 코드
```java
public class ProductTest {
    public static void main(String[] args) {
```

| 항목 | 점수 | 평가 기준 |

| 필드 선언 | 5점 | name, price, stock 3개 필드 모두 선언 |
| 기본 생성자 | 2점 | 매개변수 없는 생성자 작성 |

## 자주 하는 실수

### 1. 필드명 오타
```java
// 잘못된 예
String nmae;  // name 오타
int prcie;    // price 오타
```

### 2. 생성자에서 this 미사용
```java
// 잘못된 예
public Product(String name, int price, int stock) {
    name = name;  // 필드가 아닌 매개변수에 대입됨
}

// 올바른 예
public Product(String name, int price, int stock) {
    this.name = name;  // this로 필드 구분
}
```

### 3. toString() 형식 불일치
```java
// 잘못된 예
return name + price + stock;  // 형식이 다름

// 올바른 예
return "상품명: " + name + ", 가격: " + price + "원, 재고: " + stock + "개";
```

### 4. 반환 타입 오류
```java
// 잘못된 예
public String getTotalValue() {  // int여야 함
    return price * stock;
}

public int isAvailable() {  // boolean이어야 함
    return stock > 0;
}
```

### 5. 메서드 로직 오류
```java
// 잘못된 예
public boolean isAvailable() {
    if (stock > 0) {
        return true;
    } else {
        return false;
    }
}

// 올바른 예 (간결함)
public boolean isAvailable() {
    return stock > 0;
}
```

## 추가 학습

### Product 클래스 활용 예시

```java
public class ProductExample {
    public static void main(String[] args) {
        // 상품 객체 생성
        Product product1 = new Product("스마트폰", 800000, 10);
        Product product2 = new Product("태블릿", 500000, 0);
        
        // 정보 출력
        System.out.println("=== 상품 정보 ===");
        System.out.println(product1);
        System.out.println(product2);
        
        // 총 가치 비교
        System.out.println("\n=== 총 가치 비교 ===");
        int value1 = product1.getTotalValue();
        int value2 = product2.getTotalValue();
        System.out.println("상품1 총 가치: " + value1 + "원");
        System.out.println("상품2 총 가치: " + value2 + "원");
        
        // 판매 가능 여부 확인
        System.out.println("\n=== 판매 가능 여부 ===");
        if (product1.isAvailable()) {
            System.out.println("상품1은 판매 가능합니다.");
        }
        if (product2.isAvailable()) {
            System.out.println("상품2는 판매 가능합니다.");
        } else {
            System.out.println("상품2는 재고가 없어 판매 불가능합니다.");
        }
    }
}
```

### 실행 결과
```
=== 상품 정보 ===
상품명: 스마트폰, 가격: 800000원, 재고: 10개
상품명: 태블릿, 가격: 500000원, 재고: 0개

=== 총 가치 비교 ===
상품1 총 가치: 8000000원
상품2 총 가치: 0원

=== 판매 가능 여부 ===
상품1은 판매 가능합니다.
상품2는 재고가 없어 판매 불가능합니다.
```
