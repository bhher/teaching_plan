### 출력 결과

```
-----------

-----------

2024002 김철수   21 소프트웨어   국어:92 영어:88 수학:95 총점:275 평균:91.67
```

1. `i = 0`: `studentsArray[0] != null` → 출력 (홍길동, 총점:263, 평균:87.67)
2. `i = 1`: `studentsArray[1] != null` → 출력 (김철수, 총점:275, 평균:91.67)
3. `i = 2`: `studentsArray[2] != null` → 출력 (이영희, 총점:245, 평균:81.67)
4. `i = 3`: `studentsArray[3] == null` → 건너뜀
5. ... `i = 99`: 모두 null → 종료

## 5️⃣ 학생 수정 (updateStudent)

### 실행 흐름

```
사용자 입력: 학번 "2024001"
    ↓
findStudent("2024001") 호출 → Student 객체 찾기
    ↓
새로운 정보 입력
    이름: "홍길동" (변경 없음)
    나이: 21 (20 → 21로 변경)
    전공: "소프트웨어" (컴퓨터공학 → 소프트웨어로 변경)
    국어: 90 (85 → 90으로 변경)
    영어: 92 (90 → 92로 변경)
    수학: 88 (변경 없음)
    ↓
Student 객체의 Setter 메서드 호출
    student.setName("홍길동")
    student.setAge(21)
    student.setMajor("소프트웨어")
    student.setKor(90)
    student.setEng(92)
    student.setMath(88)
    ↓
객체의 필드 값 변경
    ↓
총점과 평균 자동 재계산
```

### 수정 전 배열 상태

| 인덱스 | Student 객체 | 학번 | 이름 | 나이 | 전공 | 국어 | 영어 | 수학 | 총점 | 평균 |
|--------|-------------|------|------|------|------|------|------|------|------|------|
| [0] | ✅ | 2024001 | 홍길동 | **20** | **컴퓨터공학** | **85** | **90** | 88 | **263** | **87.67** |
| [1] | ✅ | 2024002 | 김철수 | 21 | 소프트웨어 | 92 | 88 | 95 | 275 | 91.67 |
| [2] | ✅ | 2024003 | 이영희 | 19 | 정보통신 | 78 | 85 | 82 | 245 | 81.67 |

### 수정 과정

**입력:**
- 학번: "2024001"
- 이름(수정): "홍길동"
- 나이(수정): 21
- 전공(수정): "소프트웨어"
- 국어(수정): 90
- 영어(수정): 92
- 수학(수정): 88

**메모리 변화:**

```
수정 전:
┌───────────────────────────────┐
│   Student 객체 (studentsArray[0]) │
│   studentId: "2024001"        │
│   name: "홍길동"              │
│   age: 20  ←───── 변경        │
│   major: "컴퓨터공학" ←── 변경 │
│   kor: 85  ←───── 변경        │
│   eng: 90  ←───── 변경        │
│   math: 88                    │
│   총점: 263 ←── 자동 재계산    │
│   평균: 87.67 ←── 자동 재계산  │
└───────────────────────────────┘

수정 후:
┌───────────────────────────────┐
│   Student 객체 (studentsArray[0]) │
│   studentId: "2024001"        │
│   name: "홍길동"              │
│   age: 21  ←───── 변경됨       │
│   major: "소프트웨어" ←── 변경됨 │
│   kor: 90  ←───── 변경됨       │
│   eng: 92  ←───── 변경됨       │
│   math: 88                    │
│   총점: 270 ←── 자동 재계산됨  │
│   평균: 90.00 ←── 자동 재계산됨 │
└───────────────────────────────┘
```

**주의:** 배열의 인덱스는 변경되지 않고, 같은 Student 객체의 필드 값만 변경됩니다!

**총점 재계산:** 90 + 92 + 88 = 270  
**평균 재계산:** 270 / 3.0 = 90.00

### 수정 후 배열 상태

| 인덱스 | Student 객체 | 학번 | 이름 | 나이 | 전공 | 국어 | 영어 | 수학 | 총점 | 평균 |
|--------|-------------|------|------|------|------|------|------|------|------|------|
| [0] | ✅ | 2024001 | 홍길동 | **21** ⬆️ | **소프트웨어** ⬆️ | **90** ⬆️ | **92** ⬆️ | 88 | **270** ⬆️ | **90.00** ⬆️ |
| [1] | ✅ | 2024002 | 김철수 | 21 | 소프트웨어 | 92 | 88 | 95 | 275 | 91.67 |
| [2] | ✅ | 2024003 | 이영희 | 19 | 정보통신 | 78 | 85 | 82 | 245 | 81.67 |

**변경 사항:**
- `studentsArray[0]`의 age: 20 → 21
- `studentsArray[0]`의 major: "컴퓨터공학" → "소프트웨어"
- `studentsArray[0]`의 kor: 85 → 90
- `studentsArray[0]`의 eng: 90 → 92
- `studentsArray[0]`의 총점: 263 → 270 (자동 재계산)
- `studentsArray[0]`의 평균: 87.67 → 90.00 (자동 재계산)

```

    ↓
findStudent("2024002") 호출 → Student 객체 찾기
    ↓
배열을 순회하며 해당 Student 객체 찾기
    ↓
해당 위치를 null로 설정
    studentsArray[i] = null;
    ↓
객체는 가비지 컬렉션에 의해 제거됨
```

### 삭제 전 배열 상태

| 인덱스 | Student 객체 | 학번 | 이름 | 나이 | 전공 | 국어 | 영어 | 수학 | 총점 | 평균 |
|--------|-------------|------|------|------|------|------|------|------|------|------|
| [0] | ✅ | 2024001 | 홍길동 | 21 | 소프트웨어 | 90 | 92 | 88 | 270 | 90.00 |
| [1] | ✅ | 2024002 | 김철수 | 21 | 소프트웨어 | 92 | 88 | 95 | 275 | 91.67 |
| [2] | ✅ | 2024003 | 이영희 | 19 | 정보통신 | 78 | 85 | 82 | 245 | 81.67 |
| [3] | null | - | - | - | - | - | - | - | - | - |

### 삭제 과정

**입력:** 학번 "2024002"

**배열 상태 변화:**

| 단계 | [0] | [1] | [2] | [3] |

## 8️⃣ 핵심 개념 정리

### 배열 관리 패턴

#### 1. 등록: 빈 자리 찾기

```java
for (int i = 0; i < studentsArray.length; i++) {
    if (studentsArray[i] == null) {  // 빈 자리 찾기
        studentsArray[i] = newStudent;  // 저장
        break;
    }
}
```

**특징:**
- 항상 앞에서부터 빈 자리를 찾습니다.
- 첫 번째로 만난 `null` 위치에 저장합니다.

#### 2. 삭제: null로 설정

```java
for (int i = 0; i < studentsArray.length; i++) {
    if (studentsArray[i] != null && 
        studentsArray[i].getStudentId().equals(studentId)) {
        studentsArray[i] = null;  // null로 설정
        break;
    }
}
```

**특징:**
- 삭제 후에도 배열 크기는 변하지 않습니다.
- 인덱스는 이동하지 않습니다. (빈 공간이 생김)

#### 3. 목록: null 체크

```java
for (int i = 0; i < studentsArray.length; i++) {
    if (studentsArray[i] != null) {  // null 체크 필수!
        // Student 정보 출력 (점수, 총점, 평균 포함)
    }
}
```

**특징:**
- `null`이 아닌 요소만 처리합니다.
- 빈 공간은 자동으로 건너뜁니다.

### 총점과 평균 계산

#### 총점 계산

```java
public int calculateTotal() {
    return kor + eng + math;
}
```

**예시:**
- 국어: 85, 영어: 90, 수학: 88
- 총점: 85 + 90 + 88 = 263

#### 평균 계산

```java
public double calculateAverage() {
    return calculateTotal() / 3.0;  // 3.0으로 나누어 실수 결과
}
```

**예시:**
- 총점: 263
- 평균: 263 / 3.0 = 87.666... → 87.67 (소수점 둘째 자리까지)

### 데이터 저장 구조

```
studentsArray[인덱스] → Student 객체 → 필드 값들
         │
         │ 참조 (주소)
         ▼
    Heap 메모리
    ┌─────────────┐
    │ studentId   │
    │ name        │
    │ age         │
    │ major       │
    │ kor         │
    │ eng         │
    │ math        │
    │             │
    │ calculateTotal() → 263
    │ calculateAverage() → 87.67
    └─────────────┘
```

### 주의사항

1. **null 체크 필수**: 배열 접근 전에 항상 `null` 체크
2. **인덱스 이동 없음**: 삭제 후에도 다른 요소의 인덱스는 변경되지 않음
3. **빈 공간 발생**: 삭제하면 배열에 빈 공간(null)이 생김
4. **순차 저장**: 등록 시 앞에서부터 빈 자리에 순서대로 저장
5. **총점/평균 자동 계산**: 점수 변경 시 `calculateTotal()`, `calculateAverage()` 메서드로 자동 재계산
