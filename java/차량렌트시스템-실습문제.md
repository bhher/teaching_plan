# 차량 렌트 시스템 - 실습 문제

## 📋 문제 설명
도서 대여 시스템과 유사한 구조로 차량 렌트 시스템을 구현하세요.

---

## 1️⃣ 차량 클래스 (Vehicle)

### 필드
- `String model` : 차량 모델명
- `String manufacturer` : 제조사
- `String location` : 차량 위치
- `String plateNumber` : 차량 번호판
- `boolean available` : 대여 가능 여부

### 생성자
1. **기본 생성자**: 매개변수 없음
2. **매개변수 생성자**: `model`, `manufacturer`, `location`, `plateNumber`를 받아 초기화
   - 생성 시 `available = true`로 설정 (대여 가능 상태)

### 메서드
- **Getter/Setter**: 모든 필드에 대해 getter/setter 작성
- `boolean isAvailable()`: 대여 가능 여부 반환 (boolean 필드는 `is` 접두사 사용)
- `String toString()`: 차량 정보를 문자열로 반환
  - 형식: `"모델 : [모델명], 제조사 : [제조사], 위치 : [위치], 번호판 : [번호판], 대여여부 : [대여가능/대여불가능]"`
- `void rent()`: 차량 대여 처리 메서드 (`available = false`로 변경)

## 2️⃣ 차량 관리 클래스 (VehicleManager)

### 필드
- `ArrayList<Vehicle> vehicles` : 전체 차량 리스트
- `ArrayList<Vehicle> rentedVehicles` : 대여 중인 차량 리스트

### 생성자
- 기본 생성자에서 두 리스트 초기화
- 더미 데이터 3개 추가:
  1. "소나타", "현대", "A구역", "12가3456"
  2. "캠리", "토요타", "B구역", "34나5678"
  3. "벤츠 E클래스", "메르세데스", "C구역", "56다7890"

### 메서드

#### 1. `void showAvailableVehicles()`
- 대여 가능한 차량 목록 출력
- `vehicles` 리스트를 순회하며 `available == true`인 차량만 출력

#### 2. `boolean rentVehicle(String modelName)`
- 차량 대여 처리
- 매개변수: 대여하려는 차량 모델명
- `vehicles` 리스트를 순회하며:
  - 모델명이 일치하고 (`equalsIgnoreCase` 사용)
  - 대여 가능한 차량을 찾으면:
    - `rent()` 메서드 호출
    - `rentedVehicles` 리스트에 추가
    - `true` 반환
- 조건에 맞는 차량이 없으면 `false` 반환

#### 3. `void showRentedVehicles()`
- 대여 중인 차량 목록 출력
- `rentedVehicles` 리스트의 모든 차량 출력

#### 4. `void addVehicle(String model, String manufacturer, String location, String plateNumber)`
- 새로운 차량 추가
- 매개변수로 받은 정보로 `Vehicle` 객체 생성 후 `vehicles` 리스트에 추가

#### 5. `void removeVehicle(String modelName)`
- 차량 삭제
- 매개변수: 삭제하려는 차량 모델명
- `vehicles` 리스트를 순회하며:
  - 모델명이 일치하고 (`equalsIgnoreCase` 사용)
  - 대여 중이 아닌 차량(`available == true`)만 삭제 가능
  - 삭제 성공 시 "삭제됨", 실패 시 "삭제 안됨" 출력

#### 6. `void updateVehicle(String modelName)`
- 차량 정보 수정
- 매개변수: 수정하려는 차량 모델명
- 수정할 항목 선택 메뉴:
  1. 모델명 수정
  2. 제조사 수정
  3. 위치 수정
  4. 번호판 수정
- 해당 필드만 수정하고 리스트에 반영

#### 7. `void showVehicleInfo(String modelName)`
- 차량 상세 정보 조회
- 매개변수: 조회하려는 차량 모델명
- 모델명이 일치하는 차량의 `toString()` 출력

## 3️⃣ 실행 클래스 (RentSystem)

### 메인 메뉴
```
차량 렌트 시스템에 오신 것을 환영합니다.
1. 대여 가능한 차량 보기
2. 차량 대여하기
3. 대여 중인 차량 보기
4. 차량 추가하기
5. 차량 삭제하기
6. 차량 정보 수정하기
7. 차량 정보 조회하기
8. 종료
원하는 작업을 선택하세요 >>
```

### 각 메뉴 동작

#### 1. 대여 가능한 차량 보기
- `VehicleManager.showAvailableVehicles()` 호출

#### 2. 차량 대여하기
- 사용자로부터 모델명 입력
- `VehicleManager.rentVehicle()` 호출
- 성공 시 "차량이 성공적으로 대여되었습니다."
- 실패 시 "차량이 존재하지 않거나 대여 불가능합니다."

#### 3. 대여 중인 차량 보기
- `VehicleManager.showRentedVehicles()` 호출

#### 4. 차량 추가하기
- 모델명, 제조사, 위치, 번호판 입력 받기
- `VehicleManager.addVehicle()` 호출
- "차량 추가 완료되었습니다." 출력

#### 5. 차량 삭제하기
- 삭제할 차량 모델명 입력
- 빈 문자열 체크 후 재입력 요청
- `VehicleManager.removeVehicle()` 호출
- "삭제 완료" 출력

#### 6. 차량 정보 수정하기
- 수정할 차량 모델명 입력
- 빈 문자열 체크 후 재입력 요청
- `VehicleManager.updateVehicle()` 호출
- "수정 완료" 출력

#### 7. 차량 정보 조회하기
- 조회할 차량 모델명 입력
- 빈 문자열 체크 후 재입력 요청
- `VehicleManager.showVehicleInfo()` 호출
- "조회 완료" 출력

#### 8. 종료
- "프로그램을 종료합니다." 출력 후 프로그램 종료

## 🎯 핵심 학습 포인트

1. **ArrayList 사용**: 동적 리스트 관리
2. **객체 지향 설계**: 도메인 클래스와 관리 클래스 분리
3. **캡슐화**: private 필드와 public 메서드 사용
4. **문자열 비교**: `equalsIgnoreCase()` 대소문자 무시 비교
5. **예외 처리**: 빈 문자열 체크 및 재입력
6. **CRUD 작업**: 생성(추가), 읽기(조회), 수정, 삭제 구현
7. **상태 관리**: 대여 가능/불가능 상태 변경

---

## 💡 참고사항

- 도서 대여 시스템과 동일한 구조와 패턴 사용
- 모든 문자열 비교는 `equalsIgnoreCase()` 사용 (대소문자 무시)
- 대여 중인 차량은 삭제 불가능
- Scanner 사용 후 `nextLine()` 처리 주의
- 반복문에서 리스트 요소를 삭제할 때 주의 (Iterator 또는 인덱스 사용)
